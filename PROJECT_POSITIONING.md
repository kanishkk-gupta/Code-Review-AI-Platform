# CodeGuardian AI — Project Positioning

This document defines the canonical messaging, scope, and positioning for the CodeGuardian AI repository. All documentation, release notes, and GitHub metadata must align with these definitions to ensure honest and professional communication of the Version 1.0 capabilities.

## 1. Official Product Name
**CodeGuardian AI**

## 2. One-Line Elevator Pitch
A deterministic, multi-agent static analysis pipeline built on LangGraph and FastAPI to automate code review, architectural auditing, and security scanning.

## 3. Official Repository Description
An async-first code review API orchestrating a multi-agent static analysis workflow via LangGraph. It surfaces bugs, security vulnerabilities, complexity hotspots, and architectural smells using strict AST parsing and deterministic rule-based heuristics.

## 4. GitHub Topics/Tags
`static-analysis`, `langgraph`, `fastapi`, `code-review`, `ast`, `pydantic`, `python`, `code-quality`, `dev-tools`

## 5. Version 1.0 Feature Set
- **Async API**: FastAPI endpoints with job polling (`/review`, `/status`, `/report`).
- **Agentic Orchestration**: LangGraph state machine orchestrating highly-concurrent, parallel analysis nodes.
- **5 Deterministic Analyzers**: Bug, Security, SOLID, Complexity, and Architecture agents powered by regex, Python AST traversal, Radon metrics, and graph dependency analysis.
- **Comprehensive Reporting**: Automated generation of scored markdown and PDF reports.
- **Resilient Ingestion**: Support for GitHub repository cloning and ZIP uploads with line-preserving file chunking.
- **Data Integrity**: Strict runtime data contracts enforced by Pydantic V2.

## 6. Features Intentionally NOT Included in Version 1.0
- **LLM-Powered Confirmation**: LLM API calls are explicitly stubbed out (`NotImplementedError`) to prioritize deterministic, zero-cost analysis.
- **RAG (Retrieval-Augmented Generation)**: Vector embeddings and FAISS indexes are constructed during ingestion but are not currently queried for semantic retrieval or context enrichment.
- **Web Dashboard**: A Streamlit UI prototype exists but is not officially supported or fully wired to the backend.

## 7. Version 2.0 Roadmap
- **LLM Synthesis & Verification**: Wire OpenAI/Ollama clients to enable Phase 2 AI-confirmation, drastically reducing false positives on regex-based hits.
- **RAG Context Enrichment**: Integrate the `ChunkRetriever` to query the FAISS index, providing agents with deep, cross-file semantic context.
- **Aggressive Test-File Filtering**: Improve heuristics to silence vulnerability and complexity noise generated by unit test suites.
- **Official Web Frontend**: Complete the end-to-end Streamlit web dashboard for interactive report visualization.
